JSP

前言

不要求在日常项目中学会嵌入java代码。只需了解JSP的基本作用，原理和会看JSP代码便可。个人多用EL或JSTL书写更简洁的表达式方法

1. 简介

JSP定义

  • JSP全称Java Server Pages，是一种动态网页开发技术。主要用于实现Java web应用程序的用户界面部分
  • JSP 技术是以 Java 语言作为脚本语言的，使用JSP标签可在HTML网页中插入Java代码 格式 <% java代码 %>

JSP结构

  • 服务器是无法直接处理Jsp页面请求的，需要一个JSP引擎（JSP容器）来截取和处理Jsp请求。而Apache内嵌有JSP容器。
  • 下图显示了服务器，JSP容器，JSP文件和数据库在Web应用中所处的位置

![](PhotoMd/20200206165349306_16058.png =524x)

JSP处理过程

  1. 与其他网页一样，浏览器发送HTTP请求给服务器
  2. Web服务器识别出为JSP网页的请求，将该请求交给JSP引擎处理。
    （通过识别URL或.jsp文件来确定是否为JSP网页的请求）
  3. JSP引擎将JSP文件转化为Servlet文件(即将模板文本改为orintln()语句，将JSP元素转化为Java代码)
  4. JSP引擎将Servlet文件编译为可执行类（.class）。
  5. Web服务器的某部件将调用Servlet引擎，载入Servlet类，而原始请求也会传递给Sevlet引擎处理。
  6. Servlet引擎在处理过程中，产生HTML格式的输出，并将其内嵌于HTTP response中，上交给Web服务器。
  7. Web服务器以静态HTML网页形式将HTTP response返回给浏览器。
  8. Web浏览器处理HTTP response中的HTML网页，显示出来
  • 理解：由上可知，JSP引擎会将JSP网页转化为Servlet文件，而原始请求会交由Servlet引擎处理，再有Servlet引擎将Servlet文件和转化为HTML文件交由浏览器解释显示出来。JSP网页就相当用另一种方式来编写Servlet

2. JSP指令

  • 说明：用于设置整个JSP页面相关的属性，如网页的编码方式和脚本语言

  • 格式：<%@ directive attribute=”value” %>

指令 描述
<%@ page … %> 定义网页依赖属性，比如脚本语言、error页面、缓存需求等等
<%@ include … %> 包含其他文件 （JSP文件，HTML文件，文本文件）
<%@ taglib … %> 引入标签库的定义

Page指令相关属性

属性 描述
buffer 指定out对象使用缓冲区的大小
autoFlush 控制out对象的 缓存区
contentType 指定当前JSP页面的MIME类型和字符编码
errorPage 指定当JSP页面发生异常时需要转向的错误处理页面
isErrorPage 指定当前页面是否可以作为另一个JSP页面的错误处理页面
extends 指定servlet从哪一个类继承
import 导入要使用的Java类
info 定义JSP页面的描述信息
isThreadSafe 指定对JSP页面的访问是否为线程安全
language 定义JSP页面所用的脚本语言，默认是Java
session 指定JSP页面是否使用session
isELIgnored 指定是否执行EL表达式
isScriptingEnabled 确定脚本元素能否被使用

Include指令相关属性

属性 描述

Taglib指令相关属性

属性 描述
uri 确定标签库的位置

4. JSP内置对象

  • 说明：内置对象中包含有四个作用域对象。
内置对象 描述
pageContext PageContext类的实例
作用域：当前页面
作用：可获取其他八个内置对象的所有信息
request HttpServletRequest类的实例
作用域：一次请求，只要服务器对请求做出响应，该作用域便销毁
作用：获取HTTP头信息，cookies，HTTP方法等等,与在该作用域的相关对象
session HttpSession类的实例
作用域：同一个会话
作用：用来跟踪，获取在各个客户端请求间的会话
application ServletContext类的实例
作用域：服务器开始创建，关闭则销毁
作用：对全局对象属性数据的获取和设置
作用域比较 pageContext < request < session < application
out PrintWriter类的实例
用来在response对象中写入内容
response HttpServletResponse类的实例，可用于添加新的cookies，时间戳，HTTP状态码等等
config ServletConfig类的实例
作用：用于访问Servlet或者JSP引擎的初始化参数，比如文件路径等
page 类似于Java类中的this关键字，该对象代表着这个页面
exception exception类的对象，代表发生错误的JSP页面中对应的异常对象

EL表达式

1. 简介

  • 说明：简化JSP内置对象属性获取的书写

  • 前提：引入el.jar与standard.jar 

2. 作用

  1. 可以与 JSTL 以及 JavaScript 结合使用。
  2. 可自动执行数值转换（null 值直接以空字符串显示）
  3. 可以访问 JavaBean 中的属性、嵌套属性和集合属性
  4. 可实现算术、逻辑、关系、条件等多种运算
  5. 可以获得命名和空间
  6. 执行除法时如果除数是 0，则返回无穷大（Infinity），不返回错误。
  7. 可访问 4 种 JSP 的作用域（request、session、application、page）。
  8. 扩展函数可以与 Java 类的静态方法执行映射。

JSTL

  • JSTL标签库：https://www.runoob.com/jsp/jsp-jstl.html

1. 简介

定义

  • JSTL（Java server pages standarded tag library，即JSP标准标签库）是一个JSP标签集合，它封装了JSP应用的通用核心功能。简化JSP逻辑的书写

  • JSTL支持通用的、结构化的任务，比如迭代，条件判断，XML文档操作，国际化标签，SQL标签。 除了这些，它还提供了一个框架来使用集成JSTL的自定义标签。

  • JSTL标签所提供的功能可分为5类：核心标签，格式化标签，SQL标签，XML标签，JSTL标签

2. 应用

（1）核心标签

  • 说明：核心标签是最常用的 JSTL标签

  • 注意：引入核心标签库
    <%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>

标签 描述
<c:out> 用于在JSP中显示数据，就像<%= … >
<c:set> 用于保存数据
<c:remove> 用于删除数据
<c:catch> 用来处理产生错误的异常状况，并且将错误信息储存起来
<c:if> 与我们在一般程序中用的if一样
<c:choose> 本身只当做<c:when>和<c:otherwise>的父标签
<c:when> <c:choose>的子标签，用来判断条件是否成立
<c:otherwise> <c:choose>的子标签，接在<c:when>标签后，当<c:when>标签判断为false时被执行
<c:import> 检索一个绝对或相对 URL，然后将其内容暴露给页面
<c:forEach> 基础迭代标签，接受多种集合类型
<c:forTokens> 根据指定的分隔符来分隔内容并迭代输出
<c:param> 用来给包含或重定向的页面传递参数
<c:redirect> 重定向至一个新的URL.
<c:url> 使用可选的查询参数来创造一个URL
