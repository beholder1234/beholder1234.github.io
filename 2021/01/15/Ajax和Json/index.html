Ajax

前言

  • 简单学习Ajax的用法，Jquery/axios等会对Ajax封装，用法更简单，兼容更好。此教程用于了解Ajax的基本作用和基本用法便可。

1. 简介

  • AJAX （Asynchronous JavaScript + XML）（异步的 JavaScript 和 XML）。
  • AJAX是一种创建交互式网页应用的网页开发技术。用于完成前端对后端的响应和请求或只是单纯的控制前端页面的一些交互行为（如：鼠标点击字体变大）
  • AJAX 最大的优点是在不重新加载整个页面的情况下，可以与服务器交换数据并更新部分网页内容。（局部更新）

2. 交互对象

说明：（XMLHttpRequest对象）

  • 现代浏览器均支持XMLHttpRequest对象，与服务器数据交换，意味着可以在不重新加载整个网页的情况下，对网页的某部分进行更新。
  • XMLHttpRequest对象作为与服务器交互的对象，在交互过程中必然会出现各种状态
属性/事件/方法 描述
onreadystatechange事件 存储函数（或函数名），每当 readyState 属性改变时，就会调用该函数。
readyState属性 存有 XMLHttpRequest 的状态。从 0 到 4 发生变化。
0: 请求未初始化
1: 服务器连接已建立
2: 请求已接收
3: 请求处理中
4: 请求已完成，且响应已就绪
status属性 200: “OK”
404: 未找到页面
open(method,url,async)方法 规定请求的类型、URL 以及是否异步处理请求。
method：请求的类型；GET 或 POST
url：文件在服务器上的位置
async：true（异步）或 false（同步）
send(string)方法 将请求发送到服务器。
string：仅用于 POST 请求
setRequestHeader(header,value)方法 向请求添加 HTTP 头。
header: 规定头的名称
value: 规定头的值
responseText属性 获得字符串形式的响应数据
responseXML属性 获得XML形式的响应数据

Json

了解更多：https://www.sojson.com/json/json_index.html

1. Json简介

  • JSON是一种数据存储，传输交换格式
  • JSON数据可在前后端传递与解释，主要前后端交换数据时使用
  • JSON 是一种有规则字符串。
  • JSON 以Key/Value存储，Key支持String，Number类型；Value支持几乎所有类型
  • JSON 支持几乎所有类型的定义与读取（包括对象）

方法 实例
JSON.parse(text, reviver) json格式字符串转javascript对象，不可转译函数
text:必需， 一个有效的 JSON 字符串
reviver: 可选，一个转换结果的函数， 将为对象的每个成员调用此函数。
eval( ‘(‘+ text +’)’ ) json格式字符串转javascript对象,可转译并执行函数
text:必需， 一个有效的 JSON字符串
JSON.stringify(value, replacer, space) 将 JavaScript 对象转换为Json格式字符串,会删除javaScript对象的函数，包括key与value
value: 必需， 要转换的 JavaScript 值（通常为对象或数组）
replacer: 可选。用于转换结果的函数或数组。
space:可选
